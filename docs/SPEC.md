# Flower Desktop - 桌面养花游戏

> 桌面花圃养花游戏 — 种花、培育、收藏，没事浇浇水

## 1. 项目概述

### 1.1 基本信息
| 项目 | 内容 |
|-----|------|
| 项目名称 | Flower Desktop |
| 类型 | 桌面装饰 + 休闲养成 |
| 引擎 | Godot 4.2+ |
| 语言 | GDScript |
| 平台 | Windows / Linux |
| 显示模式 | 桌面底部悬浮窗（展示）+ 花圃窗口（游戏） |

### 1.2 核心特色
- **桌面屏保** — 把喜欢的花摆在桌面上，花会自己轻轻摇曳
- **花圃种植** — 2D俯视花圃，浇水养花，看着它慢慢长大
- **培育惊喜** — 两朵花配对培育，永远不知道下一朵是什么样
- **收集图鉴** — 养到开花就收录，看看还能发现什么新花
- **零压力** — 花不会死、不会枯萎，不玩也不会有任何损失

### 1.3 设计理念
- **休闲第一**：不强迫登录，没有惩罚，没有损失
- **惊喜驱动**：培育的结果永远有随机性，每次都不一样
- **桌面装饰**：花不只是游戏内容，也是桌面的一部分
- **收集乐趣**：图鉴里的剪影在暗示你还有多少没见过

## 2. 两个空间

游戏分为两个独立空间，各司其职。

### 2.1 桌面展示

- 悬浮在屏幕底部，半透明，不遮挡桌面
- 玩家从花园里挑选最喜欢的花摆到桌面上
- 花在桌面上做 idle 动画（摇曳、呼吸、偶尔掉花瓣）
- 纯装饰，不承载任何游戏操作
- 点击桌面上的花可以打开花圃界面

### 2.2 花圃（主游戏界面）

- 2D 俯视视角的小花园
- 格子化种植，每格种一株花
- 所有游戏操作在这里完成：浇水、培育、管理
- 打开时是一个独立窗口，关闭后回到桌面展示

## 3. 核心玩法

### 3.1 核心操作

整个游戏只有两个操作：

| 操作 | 效果 |
|-----|------|
| **浇水** | 点一下花就是浇水，推动生长进度 |
| **培育** | 选两朵开花的花配对，生成新花 |

### 3.2 生长系统

**生长进度 = 浇水次数，不依赖真实时间。**

| 阶段 | 需要 | 视觉表现 |
|-----|------|---------|
| 种子 | 2次浇水 | 小土堆 |
| 发芽 | 3次浇水 | 冒出小芽 |
| 幼苗 | 3次浇水 | 长出叶子 |
| 成株 | 4次浇水 | 茎叶丰满 |
| 开花 | 永久 | 花朵绽放，进入 idle 动画 |

- 每次浇水都能看到明显的变化
- 开花后不需要再照料，永久保持
- 不玩不会退步，花不会死

### 3.3 培育系统

培育是这个游戏的核心乐趣和惊喜来源。

#### 操作流程
1. 在花圃里点一朵开花的花
2. 选择"培育"
3. 再点另一朵开花的花
4. 两朵花中间长出一棵新芽苗
5. 给芽苗浇水几次
6. 开花揭晓结果

#### 三层结果

**混色（常见）**
- 同品种的花互相培育，颜色混合 + 随机偏移
- 红玫瑰 + 白玫瑰 → 粉色/橙粉/浅红... 每次都不一样
- 这是培育的基础体验，结果可预感但有惊喜

**跨品种（较少）**
- 不同品种的花互相培育，可能产生全新品种
- 玫瑰 + 郁金香 → 一种既不像玫瑰也不像郁金香的新花
- 具体组合表由系统定义，但玩家不知道全部配方
- 给玩家"试试看会怎样"的好奇心

**稀有花（罕见）**
- 任何培育都有极小概率触发
- 不靠特定配方，纯随机惊喜
- 触发时：花盆发光 → 全屏花瓣飘落 → 特殊音乐
- 不能被攻略，只能被遇见

#### 培育的设计原则
- 不显示基因数据、不显示概率
- 过程有期待感（芽苗颜色慢慢变化暗示结果）
- 即使是"普通"结果，颜色也每次不同
- 稀有花是纯粹的天降惊喜

### 3.4 花圃管理

**花圃布局**
- 俯视2D花圃，格子化种植
- 一开始 6-9 个种植位
- 随着收集增多，花圃可以扩展（解锁更多格子）
- 扩展本身也是一种成就感

**花的来源**
- 初始提供几种基础种子
- 后续所有新花都通过培育获得

**花圃操作**
- 点空白格子 → 种种子（从已有种子中选）
- 点花 → 浇水 / 培育 / 摆到桌面
- 开花的花可以随时用于培育

## 4. 收集与图鉴

### 4.1 图鉴系统

- 养到开花一次就自动收录
- 纯记录性质，不绑定任何奖励或解锁条件
- 首次收录时播放简短庆祝动画（花瓣飘落 + 名字浮出）

### 4.2 图鉴界面

```
┌───────────────────────────────────────┐
│           📖 我的图鉴                   │
│                                       │
│  已收集: 5/30                          │
│                                       │
│  ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐    │
│  │ 🌷  │ │ 🌹  │ │ ▓▓  │ │ 🌻  │    │
│  │ ✓   │ │ ✓   │ │剪影 │ │ ✓   │    │
│  └─────┘ └─────┘ └─────┘ └─────┘    │
│                                       │
│  ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐    │
│  │ ▓▓  │ │ 🌼  │ │ ▓▓  │ │ 🪴  │    │
│  │剪影 │ │ ✓   │ │剪影 │ │ ✓   │    │
│  └─────┘ └─────┘ └─────┘ └─────┘    │
└───────────────────────────────────────┘
```

**关键设计**：
- 未解锁的植物显示为 **剪影/轮廓**（不是锁头图标）
- 剪影暗示植物的形状，激发好奇心
- 不显示任何获取提示，让玩家自己探索
- 已收集的显示正常色彩 + 名称

## 5. 桌面展示（屏保模式）

### 5.1 桌面悬浮窗

- 位于屏幕底部，半透明
- 宽度可调节，高度紧凑
- 无边框，融入桌面
- 始终置顶但不抢焦点

### 5.2 Idle 动画

每株花有独特的 idle 动画节奏，让桌面"活"起来：

| 花的类型 | 动画风格 |
|---------|---------|
| 向日葵 | 缓慢转头追光 |
| 樱花 | 偶尔飘落花瓣 |
| 薰衣草 | 随风摆动 |
| 多肉 | 几乎静止，偶尔微微抖动 |
| 玫瑰 | 花瓣微微呼吸（缩放） |
| 稀有花 | 微光/粒子效果 |

每种花 3-5 种微动画随机循环，不是一直重复同一个动作。

### 5.3 稀有花特效

- 稀有花在桌面上有持续的微光效果
- 轻柔的粒子环绕
- 不会太抢眼，但能看出"这朵花不一样"

## 6. 视觉设计

### 6.1 整体风格
- **卡通风格**，圆润可爱
- **柔和配色**，温馨舒适
- 花圃有泥土、小石子、围栏等细节
- 不追求写实，追求可爱和辨识度

### 6.2 色彩方案
| 用途 | 颜色 | Hex |
|-----|------|-----|
| 主色调 | 草绿 | #4CAF50 |
| 泥土色 | 土棕 | #8D6E63 |
| 强调色 | 花瓣粉 | #F48FB1 |
| 花圃背景 | 浅绿 | #E8F5E9 |
| 桌面背景 | 半透明白 | #FFFFFF80 |
| 文字色 | 深灰 | #424242 |
| 稀有标识 | 金色 | #FFD700 |

### 6.3 字体
- 主字体：圆润无衬线
- 中文支持：思源黑体或类似

### 6.4 动画
- 浇水：水滴落下 + 小水花
- 生长过渡：渐变缩放
- 花朵绽放：花瓣展开
- 培育芽苗：从土里冒出
- 稀有花出现：全屏花瓣飘落 + 发光
- 新发现收录：花瓣飘落 + 名字浮出

## 10. 技术方案

### 10.1 窗口实现

**桌面展示窗口**
- Godot 悬浮窗，屏幕底部置顶
- 半透明无边框
- 只展示花的 idle 动画

**花圃窗口**
- 点击桌面花时弹出
- 独立窗口或覆盖层
- 关闭后回到桌面展示
