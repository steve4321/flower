# Flower Desktop - 桌面养花游戏

> 桌面花圃养花游戏 — 种花、培育、收藏，没事浇浇水

## 1. 项目概述

### 1.1 基本信息
| 项目 | 内容 |
|-----|------|
| 项目名称 | Flower Desktop |
| 类型 | 桌面装饰 + 休闲养成 |
| 引擎 | Godot 4.6.2 |
| 语言 | GDScript |
| 平台 | Windows / Linux |
| 显示模式 | 桌面底部悬浮窗（展示）+ 花圃窗口（游戏） |

### 1.2 核心特色
- **桌面屏保** — 把喜欢的花摆在桌面上，花会自己轻轻摇曳
- **花圃种植** — 2D俯视花圃，浇水养花，看着它慢慢长大
- **培育惊喜** — 两朵花配对培育，永远不知道下一朵是什么样
- **收集图鉴** — 养到开花就收录，看看还能发现什么新花
- **零压力** — 花不会死、不会枯萎，不玩也不会有任何损失

### 1.3 设计理念
- **休闲第一**：不强迫登录，没有惩罚，没有损失
- **惊喜驱动**：培育的结果永远有随机性，每次都不一样
- **桌面装饰**：花不只是游戏内容，也是桌面的一部分
- **收集乐趣**：图鉴里的剪影在暗示你还有多少没见过

## 2. 三个空间

游戏分为三个独立空间，各司其职。

### 2.1 桌面花瓶

- 悬浮在屏幕底部，半透明，不遮挡桌面
- 单个花瓶，玩家从花圃挑选喜欢的开花植物摆进去
- 点击花瓶打开 arrange popup，列出所有开花状态的花园植物
- 花瓶内可放置多朵花（数量无限制）
- 花在桌面上做 idle 动画（摇曳、呼吸、偶尔掉花瓣）
- 纯装饰，摆放后花不消耗
- 点击桌面上的花可以打开花圃界面

### 2.2 花圃（主游戏界面）

- 2D 俯视视角的小花园
- 格子化种植，每格种一株花
- 主要游戏操作在这里完成：浇水、种植、管理
- 点击开花植物显示操作菜单：存入仓库 / 移除
- 不再承载培育功能（移至专门培育室）
- 打开时是一个独立窗口，关闭后回到桌面展示

### 2.3 培育室（培育专用）

- 独立的专属场景，零压力
- 从花仓库选择 2 朵开花植物进行配对培育
- 培育结果直接获得新种子（无需浇水揭晓）
- 培育后花朵不消耗，仍在仓库中

## 3. 核心玩法

### 3.1 核心操作

整个游戏只有两个操作：

| 操作 | 效果 |
|-----|------|
| **浇水** | 点一下花就是浇水，推动生长进度 |
| **培育** | 在培育室选两朵花，直接获得新种子 |

### 3.2 生长系统

**生长进度 = 浇水次数，不依赖真实时间。**

| 阶段 | 浇水次数 | 视觉表现 |
|-----|---------|---------|
| 种子 | 2 | 小土堆 |
| 发芽 | 2 | 冒出小芽 |
| 幼苗 | 3 | 长出叶子 |
| 成株 | 3 | 茎叶丰满 |
| 开花 | 永久 | 花朵绽放，进入 idle 动画 |

> **从种子到开花共需 10 次浇水。** 此数值可调，调小加快节奏，调大增加期待感。

- 每次浇水都能看到明显的变化
- 开花后不需要再照料，永久保持
- 不玩不会退步，花不会死

### 3.3 培育系统

培育是这个游戏的核心乐趣和惊喜来源。

#### 操作流程
1. 进入培育室场景
2. 从花仓库（flower_storage）中选择第一朵开花植物
3. 从花仓库中选择第二朵开花植物
4. 确认配对
5. 直接获得新种子到种子背包（无需浇水揭晓）
6. 原花朵不消耗，仍在仓库中

> **培育消耗花朵：否。** 同一朵花可以反复用于培育，没有消耗。

#### 培育规则：基于植物学常识

**核心原则：同属同种才能杂交，不同科的植物不能杂交。花和多肉不能杂交。**

玩家不需要知道这些规则——他们自然会发现"把相似的花放在一起更容易出新品种"。

#### 培育组划分

植物按直觉亲缘关系分为以下培育组：

| 培育组 | 包含品种 | 玩家直觉 |
|--------|---------|---------|
| **蔷薇系** | 红玫瑰、白玫瑰、粉玫瑰、樱花 | 都是蔷薇科，花瓣层叠 |
| **百合系** | 黄郁金香、橙郁金香、紫郁金香、百合 | 都是百合科球根花卉 |
| **菊系** | 白雏菊、向日葵、康乃馨 | 都是菊类，花心明显 |
| **兰系** | 蝴蝶兰、薰衣草 | 花形优雅的特殊花 |
| **多肉系** | 观音莲、玉露、熊童子、玉龙观音 | 景天科多肉 |
| **仙人掌** | 仙人掌 | 独立一类 |

> 培育组可后期调整/扩充。

#### 三层结果

**第一层：同品种混色（常见，~70%）**
- 同品种的花互相培育，产出同品种的不同颜色
- 红玫瑰 + 白玫瑰 → 粉色玫瑰、浅红玫瑰（仍是玫瑰）
- 颜色计算：每通道 = lerp(父A, 父B, 随机权重) + rand(-20, 20)
- 这是培育的基础体验，结果可预感但有惊喜

**第二层：同培育组杂交（较少，~27%）**
- 同一培育组内的不同品种可以杂交，产出该组的新品种
- 玫瑰 + 樱花（同属蔷薇系）→ 可能有新品种
- 雏菊 + 向日葵（同属菊系）→ 可能有新品种
- 不同培育组之间 → 无法杂交，结果退回混色
- 花 × 多肉 → 完全不行，提示"这两种花差异太大"
- 系统内建杂交表，定义同组内哪些组合出什么

**第三层：稀有变异（罕见，~3%）**
- 任何培育都有 3% 概率出现基因突变
- 突变产物仍在亲本品种范围内，只是颜色极端化
- 不是"杂交出新物种"，而是"罕见变异个体"
- 触发时：全屏花瓣飘落 → 特殊音乐
- 不能被攻略，只能被遇见

> **概率分配：70% 混色 / 27% 同组杂交 / 3% 稀有变异。** 均可调。

#### 同组杂交表

**蔷薇系内部杂交：**

| 亲本A | 亲本B | 结果 |
|-------|-------|------|
| 玫瑰（任意色） | 樱花 | 牡丹 |

**百合系内部杂交：**

| 亲本A | 亲本B | 结果 |
|-------|-------|------|
| 郁金香（任意色） | 百合 | 风信子 |

**菊系内部杂交：**

| 亲本A | 亲本B | 结果 |
|-------|-------|------|
| 雏菊 | 向日葵 | 格桑花 |
| 雏菊 | 康乃馨 | 满天星 |

**多肉系内部杂交：**

| 亲本A | 亲本B | 结果 |
|-------|-------|------|
| 观音莲 | 玉露 | 玉龙观音 |
| 观音莲 | 熊童子 | 仙人掌 |

> 杂交表可随时扩充。不在表中的同组组合仍产出混色。不同组之间无法杂交。

#### 不可杂交的组合

| 组合 | 结果 |
|------|------|
| 花 × 多肉 | 提示"差异太大" |
| 花 × 仙人掌 | 提示"差异太大" |
| 蔷薇系 × 百合系 | 退回混色（按亲本A品种） |
| 蔷薇系 × 菊系 | 退回混色（按亲本A品种） |
| 任何不同培育组 | 退回混色（按亲本A品种） |

#### 稀有变异判定

| 稀有花 | 颜色条件 | 所属培育组 |
|--------|---------|-----------|
| 彩虹玫瑰 | R>200, G>200, B>200 | 蔷薇系培育时触发 |
| 暗夜曼陀罗 | R<50, G<50, B<50 | 任何培育都有极小概率 |
| 金色向日葵 | R>200, G>180, B<50 | 菊系培育时触发 |
| 月光百合 | R<80, G>180, B>180 | 百合系培育时触发 |
| 永恒之花 | 以上都不匹配时的 fallback | 任何培育 |

> 稀有花触发时，按优先级从上往下匹配。颜色阈值均可调。

#### 培育的设计原则
- 不显示基因数据、不显示概率
- 不显示培育组划分（玩家自己发现规律）
- 零压力：培育不消耗花朵，可以反复尝试
- 即使是"普通"结果，颜色也每次不同
- 稀有变异是纯粹的天降惊喜
- 玩家通过尝试自然理解："相似的花放在一起更容易出新品种"

### 3.4 花圃管理

**花圃布局**
- 俯视2D花圃，格子化种植
- 初始 **2×3 = 6 格**
- 格子间距 8px，有泥土纹理
- 随着收集增多，花圃可以扩展
- 扩展规则：**每收集 5 种新花，解锁 +3 格**（增加一行）
- 最大可扩展到 5×4 = 20 格
- 扩展本身也是一种成就感

**花的来源**
- 初始提供 3 种基础种子（红玫瑰、白雏菊、黄郁金香）
- 后续所有新花都通过培育获得
- 培育出的新花开花后，自动加入种子库，可以无限种植该品种

**花圃操作**
- 点空白格子 → 种种子（从已有种子中选）
- 点生长中的花 → 浇水
- 点开花的花 → 显示操作菜单：存入仓库 / 移除
- 开花的花不再直接在花圃用于培育，而是存入花仓库

**花仓库（flower_storage）**
- 开花植物的存储区域，容量无限制
- 所有开花植物都会移入这里
- 培育时从仓库中选择花朵
- 桌面花瓶也从这里选取植物

**花圃信息栏**
- 显示：已收集数量 / 花圃格子数 / 花仓库数量 / 种子数量

## 4. 收集与图鉴

### 4.1 图鉴系统

- 养到开花一次就自动收录
- 纯记录性质，不绑定任何奖励或解锁条件
- 首次收录时播放简短庆祝动画（花瓣飘落 + 名字浮出）

### 4.2 图鉴界面

```
┌───────────────────────────────────────┐
│           📖 我的图鉴                   │
│                                       │
│  已收集: 5/30                          │
│                                       │
│  ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐    │
│  │ 🌷  │ │ 🌹  │ │ ▓▓  │ │ 🌻  │    │
│  │ ✓   │ │ ✓   │ │剪影 │ │ ✓   │    │
│  └─────┘ └─────┘ └─────┘ └─────┘    │
│                                       │
│  ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐    │
│  │ ▓▓  │ │ 🌼  │ │ ▓▓  │ │ 🪴  │    │
│  │剪影 │ │ ✓   │ │剪影 │ │ ✓   │    │
│  └─────┘ └─────┘ └─────┘ └─────┘    │
└───────────────────────────────────────┘
```

**关键设计**：
- 未解锁的植物显示为 **剪影/轮廓**（不是锁头图标）
- 剪影暗示植物的形状，激发好奇心
- 不显示任何获取提示，让玩家自己探索
- 已收集的显示正常色彩 + 名称

## 5. 桌面展示（屏保模式）

### 5.1 桌面花瓶

- 位于屏幕底部，半透明
- 单个花瓶，可放置多朵开花植物
- 宽度可调节，高度紧凑
- 无边框，融入桌面
- 始终置顶但不抢焦点
- 点击花瓶打开 arrange popup，从花圃的开花植物中选择

### 5.2 桌面花瓶操作

- 点击花瓶 → 打开 arrange popup
- 从列表中选择花圃中已开花的植物放入花瓶
- 放入的花朵不消耗，仍在花圃/仓库中
- 可以随时更换花瓶中的花朵

### 5.3 Idle 动画

每朵花有独特的 idle 动画节奏，让桌面"活"起来：

| 花的类型 | 动画风格 |
|---------|---------|
| 向日葵 | 缓慢转头追光 |
| 樱花 | 偶尔飘落花瓣 |
| 薰衣草 | 随风摆动 |
| 多肉 | 几乎静止，偶尔微微抖动 |
| 玫瑰 | 花瓣微微呼吸（缩放） |
| 稀有花 | 微光/粒子效果 |

每种花 3-5 种微动画随机循环，不是一直重复同一个动作。每朵花独立动画，互不干扰。

### 5.4 稀有花特效

- 稀有花在桌面上有持续的微光效果
- 轻柔的粒子环绕
- 不会太抢眼，但能看出"这朵花不一样"

## 6. 视觉设计

### 6.1 整体风格
- **卡通风格**，圆润可爱
- **柔和配色**，温馨舒适
- 花圃有泥土、小石子、围栏等细节
- 不追求写实，追求可爱和辨识度

### 6.2 色彩方案
| 用途 | 颜色 | Hex |
|-----|------|-----|
| 主色调 | 草绿 | #4CAF50 |
| 泥土色 | 土棕 | #8D6E63 |
| 强调色 | 花瓣粉 | #F48FB1 |
| 花圃背景 | 浅绿 | #E8F5E9 |
| 桌面背景 | 半透明白 | #FFFFFF80 |
| 文字色 | 深灰 | #424242 |
| 稀有标识 | 金色 | #FFD700 |

### 6.3 字体
- 主字体：圆润无衬线
- 中文支持：思源黑体或类似

### 6.4 动画
- 浇水：水滴落下 + 小水花
- 生长过渡：渐变缩放
- 花朵绽放：花瓣展开
- 培育芽苗：从土里冒出
- 稀有花出现：全屏花瓣飘落 + 发光
- 新发现收录：花瓣飘落 + 名字浮出

## 8. 音效设计

### 8.1 背景音乐
- 花圃内播放轻柔的自然音乐，可关闭
- 桌面展示模式不播放音乐

### 8.2 音效列表
| 事件 | 音效 |
|-----|------|
| 浇水 | 清脆水滴声 |
| 生长阶段变化 | 轻柔的"叮" |
| 培育配对 | 温暖的和弦 |
| 培育芽苗出现 | 泥土冒出的"噗" |
