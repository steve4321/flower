- [x] 实现浇水交互：点击花 → water_count++ → 检查阶段推进 → 更新sprite

```

  点击空格 → 弹出种子列表（来自 game_state.seeds）
    → 选择种子 → 创建 Plant 实例
    → 放入 grid[plot_index]
    → 显示种子 sprite
```

## P3 - 培育系统

### 目标
能选两朵花配对培育，长出新花，有惊喜感。

### 任务清单

- [x] 实现 `scripts/autoload/gene_system.gd` — 颜色混合 + 稀有检测
- [x] 实现 `scripts/garden/breeding.gd` — 培育流程控制
- [x] 实现培育入口：点开花花 → "培育" → 进入选择模式 → 点另一朵花
- [x] 实现培育芽苗：两朵花中间生成新芽苗（占用一个空格）
- [x] 实现芽苗生长：浇水几次后开花揭晓结果
- [x] 实现三层结果逻辑：
  - 同品种混色（RGB 插值 + 随机偏移）
  - 跨品种杂交（查杂交表生成新品种）
  - 稀有花（极小概率触发）
- [x] 实现开花揭晓动画（花瓣展开 + 颜色渐变显示）
- [x] 实现稀有花特效（全屏花瓣飘落 + 发光）
- [x] 实现"新发现"判定 + 通知

### 核心逻辑

```
培育流程:
  选择 parent_a (FLOWERING) → 选择 parent_b (FLOWERING)
    → 检查有空格
    → 创建芽苗 Plant (stage=SEED, color=隐藏/灰色)
    → 放入空格
    → 玩家浇水 N 次
      → 每次浇水，芽苗颜色微微变化（暗示最终结果）
    → 最后一次浇水 → 开花
      → 计算最终颜色（mix_colors）
      → 检查跨品种表 → 可能是新品种
      → 检查稀有花 → 极小概率
      → 开花揭晓动画
      → 如果是新发现 → 触发庆祝 + 图鉴收录

颜色混合:
  child.r = lerp(a.r, b.r, randf()) + randf_range(-20, 20)
  child.g = lerp(a.g, b.g, randf()) + randf_range(-20, 20)
  child.b = lerp(a.b, b.b, randf()) + randf_range(-20, 20)

稀有检测:
  randf() < 0.03 → 检查颜色范围 → 匹配稀有花
```

## 精灵制作备注

**尺寸建议**：
- 花圃内植物：64×64（开花可适当放大）
- 桌面展示植物：64×80（竖向稍高）
- 图鉴缩略图：32×32

**风格统一**：
- 卡通圆润风格
- 描边清晰（桌面小尺寸需要辨识度）
- 开花阶段最精细，种子/发芽阶段可简化

**着色方案**：
- 每种花只做一套"白色/灰色底"精灵
- 运行时通过 `sprite.modulate = Color(r/255, g/255, b/255)` 动态着色
- 这样不同颜色变体不需要额外精灵

**剪影生成**：
- 开花精灵 → 灰度 → 阈值处理 → 纯黑轮廓
- 可脚本化批量生成

## 临时替代方案

在正式美术资源就位前，开发阶段可用以下替代：

| 正式资源 | 临时替代 |
|---------|---------|
| 植物精灵 | 彩色方块 + 文字（"种子"/"发芽"/...） |
| 花圃背景 | 纯色 + Grid绘制 |
| 特效动画 | 闪烁/缩放 tween |
| 按钮图标 | Emoji 或 文字按钮 |
| 音效 | 无声（后期统一添加） |
| 剪影 | 灰色方块 + "?" |

这样 P0-P3 可以纯代码推进，不阻塞在美术资源上。
